# 🎮 动态配置系统使用指南

## 概述

游戏已升级为**完全动态配置系统**，支持玩家自定义角色、房间、技能等所有参数，同时 AI 可以通过行为自动学习新技能。

---

## 📋 系统限制

为了控制 Token 消耗和系统稳定性：

| 项目 | 限制 | 说明 |
|------|------|------|
| **角色数量** | 最多 5 个 | 包括默认角色 |
| **房间数量** | 最多 10 个 | 包括必需的中心房间（客厅） |
| **预设数量** | 2 个 | 默认预设 + 空白配置 |
| **Token 增加** | <30% | 相对默认配置 |

## 🎮 快速开始

### 1. 游戏启动

打开游戏时，会看到**全屏设置面板**：

```
┌─────────────────────────────┐
│   🏠 公寓生活模拟器          │
│   配置游戏设置并开始游戏     │
├─────────────────────────────┤
│ ⚙️ 预设  │ 👥 角色配置 │ 🏠 房间配置 │
├─────────────────────────────┤
│ [预设卡片显示在这里]         │
│ [选择一个预设以继续]         │
├─────────────────────────────┤
│ API Key: [          ]        │
│ [开始游戏]                   │
└─────────────────────────────┘
```

### 2. 选择预设

**两个预设可选：**

- **默认：惠舞/三玖/五月** - 推荐首次游玩
- **空白配置** - 完全自定义

点击卡片选中，已选预设会高亮显示。

### 3. 自定义配置

#### 👥 角色配置标签页

**添加角色**：
- 点击 "+ 添加角色" 按钮
- 输入名字、选择颜色、设置职业等
- 最多添加到 5 个角色为止

**编辑角色字段**：
- **名字** - 角色显示名称
- **颜色** - 在日志中的显示颜色（支持颜色选择器）
- **职业** - 角色职位（如 "AI工程师"）
- **月薪** - 月收入（数字）
- **性格描述** - AI 行为参考

#### 🏠 房间配置标签页

**添加房间**：
- 点击 "+ 添加房间" 按钮
- 输入房间名称和描述
- 最多添加到 10 个房间为止

**编辑房间字段**：
- **名字** - 房间显示名称
- **ID** - 系统标识（自动生成，不可修改）
- **描述** - 房间的文字描述

### 4. API Key 配置

在 **"输入 DeepSeek API Key"** 框中输入：

- ✅ 留空：使用模拟模式（预定义的模拟 AI 回复）
- ✅ 输入 Key：连接真实 DeepSeek API

### 5. 开始游戏

点击 **"开始游戏"** 按钮：
- 设置面板消失
- 游戏画面加载
- 角色和房间按配置初始化

## ⚙️ 游戏中编辑

### 打开编辑器

游戏运行中，点击右侧面板的 **"⚙️ 配置编辑器"** 按钮：

```
┌─────────────┐
│ ⚙️ 配置编辑器 │
├─────────────┤
│ 👥 角色 │ 🏠 房间 │
├─────────────┤
│ [编辑内容]   │
└─────────────┘
```

抽屉从右侧滑出，显示可编辑的内容。

### 👥 角色编辑

### 修改颜色

- 点击颜色选择器拖动，HEX 输入框实时同步
- 直接在输入框键入色号（`#RRGGBB` 格式），颜色选择器实时同步
- 修改立即反映在日志颜色中

**管理技能**：
- 显示当前所有技能（可点击删除）
- 输入新技能名称，按 Enter 添加
- 示例：输入 "Python编程"

**调整关系度**：
- 与其他角色的关系度显示为滑块
- 范围：-100（敌对）~ +100（亲密）
- 拖动滑块实时调整

### 🏠 房间编辑

**修改描述**：
- 输入框修改房间描述
- 实时生效

**管理物品**：
- 列出房间所有物品
- 每个物品后有 "删除" 按钮
- 输入新物品名称，按 Enter 添加

## 🤖 AI 技能学习

### 工作原理

当 AI 生成行为时，可以返回 **skill_changes** 字段：

```json
{
  "character": "惠舞",
  "action": "看了美食教程，学到了新菜谱...",
  "skill_changes": {
    "learn": ["芒果布丁制作"],
    "forget": []
  },
  "stat_changes": { "mood": 5 }
}
```

游戏会自动：
1. 检测 skill_changes 字段
2. 在角色技能列表中添加新技能
3. 在编辑抽屉中显示
4. 在日志中记录学习过程

### 示例场景

**场景1：观看美食节目**
```
AI返回：
{
  "action": "三玖在电视上看到了日式料理教程，特别是那份完美的章鱼小丸子...",
  "skill_changes": {
    "learn": ["日式烹饪"]
  }
}

结果：三玖获得 "日式烹饪" 技能
```

**场景2：与朋友讨论绘画**
```
AI返回：
{
  "action": "五月和三玖讨论构图技巧，三玖耐心讲解了黄金分割...",
  "interaction_with": "三玖",
  "skill_changes": {
    "learn": ["绘画基础"]
  }
}

结果：五月获得 "绘画基础" 技能
```

## 💾 存档与恢复

### 保存游戏

点击右侧面板的 **"保存"** 按钮保存到 3 个存档槽位中的任一个。

**注意**：
- 当前配置自动保存
- 下次读档时自动恢复
- 包括已学习的所有技能

### 读取游戏

点击 **"读取"** 按钮恢复之前的存档。

## ⚡ 高级技巧

### 1. 优化 Token 消耗

- 保持角色在 3-5 个之间（默认 3 个最优）
- 保持房间在 8-10 个之间
- 简洁的职业和性格描述

### 2. 激励 AI 学习技能

在自定义角色的 "性格描述" 中添加：
```
"性格：学习欲强，经常看教程，喜欢尝试新事物"
```

这样 AI 更容易在行为中包含 skill_changes。

### 3. 创建有趣的互动

设置相反的初始关系度：
```
Alice ↔ Bob: -50（冲突设定）
```

AI 会生成更富戏剧性的互动。

### 4. 监控技能学习

定期打开编辑抽屉查看 👥 角色标签页，观察技能列表的变化。

## 🐛 故障排查

### 问题：设置面板不显示

**解决**：
- 清除浏览器缓存
- 检查浏览器控制台是否有错误
- 刷新页面重试

### 问题：修改后无效果

**解决**：
- 确保在正确的标签页编辑
- 检查是否超过了限制（5角/10房）
- 修改房间物品时检查房间 ID 是否正确

### 问题：AI 没有学习技能

**解决**：
- 确保 AI Key 正确配置（或使用 mockAI）
- 检查 AI 返回是否包含 skill_changes 字段
- 查看日志中是否显示 "学会了新技能" 消息

### 问题：颜色改了但没显示

**解决**：
- 颜色选择器与 HEX 输入框会实时双向联动，无需额外操作
- 若仍无效，关闭编辑器再重新打开
